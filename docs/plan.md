# Quiz Game 구현 작업 계획 (v2)

## 문서 목적

- `subject.md`의 필수/보너스 요구사항을 구현 가능한 작업 단위로 분해한다.
- 각 단계를 "구현 + 검증 + 커밋"으로 완료할 수 있게 만든다.
- 공통 입력/예외 처리 기준은 한 곳에 모아두고, 기능 단계에서는 참조 방식으로 적용한다.

---

## 작업 단위 분해 원칙

- 한 단계에서 한 가지 핵심 기능만 완성한다.
- 단계별 완료 기준은 실행으로 검증 가능해야 한다.
- 브랜치 필수 단계(퀴즈 풀기)는 기능 브랜치에서 작업 후 `--no-ff` 병합한다.
- 입력 검증 로직은 공통 유틸로 재사용하고, 기능별 유효 범위만 각 단계에서 정의한다.

---

## 공통 입력/예외 처리 기준 (참조)

아래 규칙은 숫자 입력이 필요한 모든 기능에서 공통으로 적용한다.

### 재입력 케이스

- 입력 앞뒤 공백을 제거한다. (`strip()`)
- 빈 입력(Enter만) 시 안내 메시지 후 재입력 흐름으로 복귀한다.
- 숫자 변환 실패(예: `abc`) 시 안내 메시지 후 재입력 흐름으로 복귀한다.
- 허용 범위 밖 숫자 입력 시 안내 메시지 후 재입력 흐름으로 복귀한다.

### 안전 종료 케이스

- `KeyboardInterrupt`(`Ctrl+C`) 발생 시 안내 메시지 출력 후 저장 가능한 상태를 저장하고 안전 종료한다.
- `EOFError` 발생 시 안내 메시지 출력 후 저장 가능한 상태를 저장하고 안전 종료한다.

### 파일 예외 케이스

- 파일이 없으면 기본 퀴즈 데이터를 사용한다.
- 파일이 손상되었으면 안내 메시지를 출력하고 기본 데이터로 복구(또는 초기화)한다.
- 파일 관련 예외의 실제 처리 구현은 Step 4에서 담당한다.

## 작업 단위 목록

### 0) 프로젝트 초기화

**목표**

- 실행 시작점과 기본 파일 구조를 만든다.

**작업**

- `main.py`, `quiz.py`, `quiz_game.py`, `README.md`, `.gitignore` 생성
- 기본 실행 진입 로직(앱 시작/종료 메시지 수준) 구성

**완료 기준**

- `python main.py` 실행 시 프로그램이 시작되고 종료된다.

---

### 1) `Quiz` 클래스 구현

**목표**

- 문제 1개를 표현하는 객체 모델을 완성한다.

**작업**

- 속성: `question`, `choices(4개)`, `answer(1~4)`, `hint(보너스 대비)`
- 메서드: 문제 출력, 정답 판정, 직렬화(`to_dict`), 역직렬화(`from_dict`)

**완료 기준**

- `Quiz` 인스턴스 생성/출력/정답 확인/딕셔너리 변환이 정상 동작한다.

---

### 2) 기본 퀴즈 데이터 구성

**목표**

- 주제 기반 퀴즈를 최소 5개 이상 확보한다.

**작업**

- 문제/선택지 4개/정답/힌트 포함 데이터 작성
- 기본 데이터 생성 함수 준비

**완료 기준**

- 초기 실행 시 최소 5개 퀴즈가 메모리에 준비된다.

---

### 3) `QuizGame` 클래스 골격 + 메뉴 루프

**목표**

- 게임 전체 제어 구조를 만들고 메뉴 이동을 가능하게 한다.

**작업**

- 속성: `quizzes`, `best_score`, `history`
- 메서드 틀: 메뉴 출력, 선택 분기, 종료
- 메뉴 입력 처리에 공통 입력/예외 처리 기준을 적용한다. (참조 섹션)
- 메뉴 유효 범위(예: `1~5`)를 정의하고 범위 검증을 연결한다.

**완료 기준**

- 메뉴가 반복 출력되고, 각 메뉴 번호가 해당 메서드로 이동한다.
- 잘못된 메뉴 입력 시 안내 메시지 후 재입력이 동작한다.

---

### 4) 파일 저장/불러오기 (`state.json`)

**목표**

- 재실행 시 데이터가 유지되도록 영속성을 완성한다.

**작업**

- `state.json` 로드/저장 구현 (UTF-8)
- 파일 없음 케이스: 기본 데이터 사용
- 파일 손상/입출력 오류 케이스: 안내 후 복구(초기화) 처리
- 안전 종료(`Ctrl+C`, `EOF`) 시 저장 흐름 연결

**완료 기준**

- 퀴즈 추가 또는 점수 갱신 후 재실행해도 데이터가 유지된다.
- 파일 없음/손상 상태에서도 프로그램이 실행 가능하다.

---

### 5) 퀴즈 풀기 기능 (브랜치 필수 단위)

**목표**

- 실제 게임 플레이 흐름을 완성한다.

**작업**

- 기능 브랜치 생성: 예) `feature/play-quiz`
- 문제 출제/정답 입력/정오답 피드백/최종 결과 출력
- 퀴즈 없음 케이스 처리
- 정답 입력 처리에 공통 입력/예외 처리 기준을 적용한다. (유효 범위: `1~4`)

**완료 기준**

- 한 라운드 플레이가 정상 종료되고 점수가 계산된다.
- 플레이 종료 직후 `best_score`가 `state.json`에 저장되고 재실행 후에도 유지된다.

**브랜치/병합 기준**

- `feature/play-quiz`에서 작업 완료 후 `main`으로 `--no-ff` 병합

---

### 6a) 퀴즈 추가 기능

**목표**

- 사용자가 새로운 퀴즈를 등록하고 데이터를 확장할 수 있게 한다.

**작업**

- 문제/선택지 4개/정답 번호/힌트 입력받기
- 입력 처리에 공통 입력/예외 처리 기준을 적용한다. (정답 번호 유효 범위: `1~4`)
- 추가 직후 `state.json`에 저장 반영

**완료 기준**

- 추가한 퀴즈가 재실행 후에도 유지된다.
- Git: 퀴즈 추가 기능 완성 후 커밋한다.

---

### 6b) 퀴즈 목록 기능

**목표**

- 사용자가 저장된 퀴즈를 확인할 수 있게 한다.

**작업**

- 전체 퀴즈 목록 출력 (번호, 문제 표시)
- 퀴즈가 없는 경우 안내 메시지 처리

**완료 기준**

- 6a에서 추가한 퀴즈가 목록에 정상 출력된다.
- 빈 목록 상태에서도 프로그램이 죽지 않고 안내 메시지를 출력한다.
- Git: 퀴즈 목록 기능 완성 후 커밋한다.

---

### 7) 점수 기능

**목표**

- 최고 점수 갱신과 조회를 완성한다.

**작업**

- 플레이 결과와 `best_score` 비교/갱신
- 점수 확인 메뉴 구현
- 아직 플레이 기록이 없는 경우 안내 처리

**완료 기준**

- 최고 점수 갱신/조회가 파일 기반으로 일관되게 동작한다.

### 8) 보너스 기능 묶음 (선택)

**목표**

- 선택 과제를 반영해 완성도를 높인다.

**작업**

- 퀴즈 삭제 + 저장 반영
- 점수 히스토리 저장 (날짜/시간, 문제 수, 점수)

**완료 기준**

- 보너스 항목이 각각 독립적으로 동작하고 저장과 연동된다.

### 10) Git 제출 요건 마감

**목표**

- 과제의 Git/GitHub 체크 항목을 모두 충족한다.

**작업**

- 의미 있는 커밋 10개 이상 유지
- 브랜치 생성/병합 기록 최소 1회 이상 확인
- clone/pull 실습
  1. 미션 저장소를 별도 로컬 디렉터리에 `clone`한다.
  2. 복제 저장소에서 간단한 변경 후 `commit -> push`한다.
  3. 기존 로컬 작업 디렉터리에서 `pull`한다.
  4. `git log` 또는 파일 내용으로 반영 여부를 확인한다.

**완료 기준**

- `git log --oneline`에서 의미 있는 커밋 10개 이상을 확인할 수 있다.
- `git log --all --oneline --graph`에서 브랜치 생성/병합 기록을 확인할 수 있다.
- clone 디렉터리와 기존 작업 디렉터리 양쪽에서 pull 반영 내용이 일치한다.

## 체크포인트

- 각 단계 종료 시 수동 실행 테스트를 먼저 한다.
- 입력/저장/복구/예외 흐름은 기능 추가 때마다 회귀 테스트한다.
- 커밋 메시지는 "무엇을 왜 바꿨는지"가 드러나게 작성한다.

## 모듈별 최소 테스트 체크리스트

- 입력 검증: `""`, `"abc"`, `" 2 "`, `"9"` 입력에서 안내 메시지/재입력 흐름 확인
- 종료 검증: `Ctrl+C`, `EOF` 발생 시 저장 후 안전 종료 확인
- `Quiz` 모델: `is_correct` 정답/오답과 `to_dict` -> `from_dict` 왕복 일치 확인
- 저장/로드: 파일 없음/정상 JSON/손상 JSON 각각의 복구 정책 확인
- 퀴즈 플레이: 퀴즈 없음/정상 라운드/중간 종료 시 저장 및 종료 동작 확인
- 보너스 기능: 랜덤/문제 수/힌트 감점/삭제/히스토리 기능 간 충돌 없음 확인
